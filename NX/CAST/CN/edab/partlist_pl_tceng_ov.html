Teamcenter Integration 零件明细表概述

同步和装配导航器

装配导航器中的装配结构是零件明细表的驱动程序。当装配导航器中的装配结构更新时，NX 中的零件明细表也会随之更新。您直接在零件明细表中所做的更改被视为“制图”应用模块中的局部更改。当更改了零件明细表中的某单元格或某项目后，如数量，该数量在装配导航器中并不更改。对引用零件明细表中项目的属性值的单元格进行的更改也被视为“制图”中的局部更改，并不反映在 Teamcenter 中。

当零件明细表所表示的结构(包括属性)与装配导航器中当前显示的结构不同时，就认为该零件明细表过时了。这不包括对零件明细表进行的手工输入，这些输入不影响装配导航器中的值，并且不被视为过时数据。

Teamcenter Integration 的默认零件明细表

要同步零件明细表和 Teamcenter Integration 数据库之间的标注值，您在 NX 中创建的初始零件明细表必须包含引用 CALLOUT 属性的一列。这样可以防止初始创建零件明细表期间 Teamcenter Integration 序列号被覆盖。

在使用 Teamcenter Integration 时，随软件一起提供的默认零件明细表包含以下列，它们包括这种必要的引用：

Teamcenter Integration 中的标注和序列号

如果正在运行 Teamcenter Integration，并在 NX 中编辑装配结构，则零件明细表中的标注将更新为 PSE 中的序列号。
