Editing code to create an application

While you can use the Journal utility and Block Styler to quickly generate NX-compliant code, simply concatenating output files from these tools does not result in a usable application. You must edit the concatenated code in order to create a runnable application that will meet your users' needs. The languages supported by NX Open are flexible and powerful, and the kinds of changes and additions needed will vary widely depending on your application's requirements and the goals of your users. However, certain modifications are required in almost every instance:

  • You must create a new Main() routine to call the subroutines in your application.

    In the output from the Journal utility and Block Styler, each separate file is defined by a single Main() routine. When you combine several code output files, you need to redefine each component Main() as a subroutine, and create a new Main() routine that calls these routines.

  • Enable any custom dialog boxes.

    If your application uses custom dialog boxes, you need to modify the code to ensure that it calls the appropriate custom dialog box at user interaction points.

  • Add functionality as needed.

    When you record a journal, information specific to the model data in NX at the time is saved with the journal file. Examples of such information include file paths and geometry selection. In order to create a general-purpose application, you need to remove or replace any model-specific code to make the application usable with a wide variety of model data. For example, you may want to replace hard-coded file paths with a variable, or replace recorded geometry selection steps with a simple Try loop.

    You can also edit your application to include standard NX functionality not captured in the initial journals.
