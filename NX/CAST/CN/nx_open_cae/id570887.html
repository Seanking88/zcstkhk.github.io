Import namespaces and create session references

The Auto_Mesh2D_ans.vb answer file is provided for comparison with your application file. If you get stuck while editing, open the answer file in Visual Studio so you can tab between the two files to locate errors. You can also use WinMerge to highlight the differences between your application file and the answer file.

If you are using Visual Studio, you will be presented with auto completion options as you type. Make sure that you are selecting the appropriate item that matches the one shown in the training material, because there are many similar items presented. This will save debugging time once you are finished.

Namespaces must be imported to be used. The System, NXOpen and NXOpen.UF import statements are created by the wizard. For this project, you need the NXOpen.CAE, NXOpen.Utilities, NXOpen.UF.Obj, and NXOpenUI namespaces. Add the highlighted lines following the Imports NXOpen.UF statement.

Note

The code additions can be cut and pasted into your application file to help reduce typographical errors.

 
Option Strict Off
Imports System
Imports NXOpen
Imports NXOpen.UF
    Imports NXOpen.CAE
Imports NXOpen.Utilities
Imports NXOpen.UF.UFObj
Imports NXOpenUI ' Added for mesh size prompt <<<<<<

Move the following Dim statements from inside the Main() routine to just after the Module statement, and add the highlighted lines. These references are now created at the module level (before Main() ).

Remove the Dim theUI As UI = UI.GetUI() statement from your journal file. 

Module Auto_Mesh2D
    Dim theSession As Session = Session.GetSession()
    Dim theUfSession As UFSession = UFSession.GetUFSession()
    Dim theLW As NXOpen.ListingWindow = theSession.ListingWindow()
    Dim theUI As NXOpen.UI = NXOpen.UI.GetUI
