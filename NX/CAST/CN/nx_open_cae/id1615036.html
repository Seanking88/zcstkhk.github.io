使用 NX CAE 数据模型中的对象

  • NX CAE 架构中的数据层级比NX CAD 架构中的数据层级更为深入。

    • 在 NX CAD 架构中，您从NX CAD 部件类的实例中获取大部分对象，比如关于体与曲线的信息。

    • 在 NX CAE 架构中，您从各种不同的对象中获取 CAE 对象，例如，考虑 MeshManager 类：

      • 您可以You can obtain a MeshManager instance from the MeshManager method from one of the FEModel classes, such as BaseFEModel or FEModel.

      • The FEModel class itself is instantiated from one of the CAEPart classes, such as BaseFemPart or FemPart.

      获取网格管理器的代码示例如下：

      Dim theSession As Session = Session.GetSession()
	Dim basePart As BasePart = Session.Parts.BaseWork
	Dim baseFEMPart As CAE.BaseFemPart = CType(BasePart, CAE.BaseFemPart)
	Dim myFEModel As FEModel = baseFEMPart.BaseFEModel
	Dim myManager As MeshManager = myFEModel.MeshManager

  • The relationships between prototype and occurrence objects is different in the NX CAD architecture than in the NX CAE architecture.

    • In the NX CAD architecture, both occurrence and prototype objects are in the same class.

    • In the NX CAE architecture, occurrence and prototype objects are in different classes but inherit from the same abstract class.

      Note

      For queries, both prototype and occurrence objects implement the same interfaces.

  • In the NX CAE architecture, you can only edit prototype objects. Builder-type methods, such as Mesh2dBuilder and ElementCreateBuilder, can only operate on the concrete classes, not on the interface classes.

  • In the NX CAE architecture, if you do not cast a pointer to the appropriate type of object (a prototype object), then your code will fail to compile. In the NX CAD architecture, if you do not ask an object for its prototype, then your code will compile but will fail to run.
