Compare your application to the target

Siemens PLM Software provides an example of the completed bottle mesh application, named bottleMeshAppAns.vb, that you can use as a control file to check your work.

Before proceeding, open your application and the completed application file in a differencing and merging tool.

There are times when extra information can be recorded in the journal files due to the rotating the model or things that are selected during journal recording. Use the file differencing tool to change any lines that show up as errors when you try to run the edited code. The changes will include line removals, and additions.

Tip

If you are using WinMerge: Start WinMerge and choose File→Open. In the Select Files or Folders dialog box, specify bottleMeshAppAns.vb on the left side and bottleMeshApp.vb on the right.

Things to notice:

  • There may be options that are set in the application file side, but not in the control file. Remove those additional lines from the application file. Also, delete any duplicate occurrences of dialog box options.

  • Ensure that the index values for undo mark, expressions, and so on are consistent. Where inconsistencies occur, merge the values from the control file into your application file.

  • There may be some lines in the application file that begin with Dim objects1, Dim cAEBody1, or Dim cAEFace. Remove these from your application file.

  • The answer file has quite a few statements in it for debugging purposes that were not copied into your application file. These lines begin with theLW.WritelLine, and are used to determine how far program execution got when there are errors. Uncomment these lines in order to see them in action.
