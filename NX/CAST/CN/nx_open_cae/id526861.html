Testing and implementation

Debugging in Visual Studio

Microsoft Visual Studio includes debugging tools you can use to debug your NX Open application.

Note

Visual Studio Express does not support debugging for DLLs that run through an external application. If you are using Visual Studio Express to debug NX Open applications, you need to implement and test the application manually.

To debug journal applications using Visual Studio:

  1. Create a Visual Studio project and build the .vb file into a managed DLL using the GetUnloadOption method.

  2. Ensure that the Configuration property (on the Property Pages window) is set to Debug.

  3. Choose Project→Properties→Debug→Start External Program, and specify the location of the NX executable.

  4. Set breakpoints in your application.

  5. Choose Debug→Start.

    Visual Studio launches NX.

  6. To run the DLL, choose File→Execute→NX Open.

Manual implementation and testing

To implement your application and test it interactively, you can create a custom button in NX and define your application as the button action. You can then add the command to a menu or toolbar.

Tip

When testing an application interactively, you may find it useful to add message boxes to act as breakpoints at various stages of the application. For example, if you add the following to your script:

' -----------------------------------------------------
' Added MessageBox just to stop execution - breakpoint
'  Have to be in Simulation by here...
' -----------------------------------------------------
messagebox.show("Just before creating FEM file", "Warning", MessageBoxButtons.OK)

The software displays the warning message “Just before creating FEM file.” Click OK to continue to executing the application. If the application fails, the last message shown can help you isolate problems in the code.
