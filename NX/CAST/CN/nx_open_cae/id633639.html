Start the application

You will run the application as a journal file and as a DLL.

To run the application as an NX Journal, select Tools→Journal→Play. Browse to locate your VB file. Run the journal and fix any errors by comparing your file with the answer file supplied with this course. (Auto_Mesh2D_ans.vb)

Note

If you have made any mistakes in entering the code, the application will not run. NX will issue an error message that points to the offending line(s). Find that location in the application file and compare it to the code that was presented in the training material. Make the change to your code and run the application. Repeat this process as necessary. The length of time to complete this process will depend on the number of mistakes there are in your code.

To run the application as a DLL, make sure that you have closed all parts and started over with the opening of the part and creation of the FEM file shown above.

In order to build the application as a DLL in Visual Studio, you will need to give your setup the properties for your project. In the Solution Explorer, right-click NX8_Open_VB_Wizard and select Properties. Select the Application tab, and set the name for both the Assembly name property and the Root Namespace property to Auto_Mesh2D.

To build the project within Visual Studio, right-click on your project and select build. Check for errors in the lower window of Visual Studio. Your DLL will be located in the bin/release or bin/debug directory within your project folder structure. Within NX, select File→Execute→NX Open and select the .DLL. Note that there may be filename length limitations, so you may need to move the DLL to a directory structure that is shorter than the default location for Visual Studio.

The application should run the same way for both methods of execution.
