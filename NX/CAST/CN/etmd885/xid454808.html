Part Modules in the Part Navigator

Part Modules appear in the Part Navigator as features divided into three collectors. When you first create a Part Module, only the Inputs collector contains objects; the Work and Outputs collectors are empty.

Inputs

This collector contains extractions of the geometry objects and expressions that you select from the part when you create the Part Module. It is what isolates the Part Module from the upstream part. These objects are the parent references for the features that you create later and that appear in the Work collector.

Work

This collector contains all the new features and expressions you create in a Part Module. Features you create can be based only on the extracted geometry and expressions in the Inputs collector.

Outputs

This collector contains extractions of those features and expressions you specify from the Work collector. Downstream features of the main part file are based only on the features in this collector and not the Work collector. It is what isolates the Part Module from the downstream part. When a Part Module is deactivated, the objects in its Outputs collector are the only objects displayed for the Part Module feature; essentially, once the work is done, a Part Module is its Outputs. All the objects in the Inputs and Work collector are hidden.
