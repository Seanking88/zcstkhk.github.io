Save As overview

You can copy a part, assembly, drawing, and so on and save it with a new name by choosing File→Save As to display the Save Part File As dialog box. The item copied is the current working item (displayed item).

Note

The item must exist in the Teamcenter database to enable Teamcenter Integration to perform a save as operation on the item.

Overview

You can perform Save As on a master or a non-master item. The actions available are different for master and non-master items. To use Save As, you:

  1. Make the item the current working item.

  2. Select the save as action.

  3. Assign or enter the item number, revision, or name depending upon the save as action and the item (master or non-master).

  4. Browse to the folder where the part will be saved (depending upon the save as action).

  5. Change attributes, if necessary.

Save as actions

The following save as actions are available for master items:

  • Create New Revision

    Creates a new revision of an existing item. The item number stays the same and the revision is automatically set to the next one in sequence. You can change this by using Assign or entering a different value.

  • Save As New Item

    Creates a new item from the existing item. The item number and revision are automatically set to the appropriate values defined at your site. You can change this by using Assign or entering different values. You can also select the folder where the new item is placed.

  • Save As New Item Type

    Creates a new item and item type from the existing item. This functionality is the same as Save As New Item (above) except you also get to select the new item type from the item types available at your site.

    Note

    This option is available only when the Allow save as new item type customer default is selected.

  • Save As Alternative Representation

    Creates an alternate representation of the existing item. The item number and revision are the same and the name is automatically set to the appropriate value defined at your site. The name is appended with the alt rep value, but you can change it by using Assign or entering different values.

    Note

    Save As Alternative Representation is not available for Simulation items.

The following save as actions are available for non-master items:

  • Save As

    Creates a new non-master from the existing non-master item. The item number and revision stay the same and the name is automatically set to the appropriate value defined at your site. You can change this by using Assign or entering a different value.

  • Save As New Item

    Creates a new master item from the existing non-master item. The item number and revision are automatically set to the appropriate values defined at your site. You can change this by using Assign or entering different values. You can also select the folder where the new item is placed.

The following customer default (File→Utilities→Customer Defaults→Teamcenter Integration for NX→General, General page) defines the default save as option:

  • Save as Action

Attributes

The Attributes button on the Save Part File As dialog box allows you to view the description, number, revision, type, name, and any other attributes that are applicable to the existing item. You can select and change the attributes based on the permissions at your site.

Projects

The Projects button opens the Assign to Project dialog box which displays the available projects in Teamcenter. The projects displayed are those that have you as a member. You can select one or more projects for the new item.

Alternate IDs

If alternate IDs are defined for the item, the Alternate ID button is displayed to allow you to select alternate IDs.

Performing a Save As on a structure only assembly with a position override

When you load an assembly as structure only (Structure only is checked on) with a position override on a component in the top level you are going to Save As and the customer default Synchronize Assembly Arrangements is enabled, the override is lost in the new assembly (in the PSE). You are not informed that the override did not occur.

To prevent this problem, perform one of the following:

  • Load all components before performing the Save As operation.

  • Do not load the assembly as structure only (Structure only not checked). Open the assembly, then save it.
