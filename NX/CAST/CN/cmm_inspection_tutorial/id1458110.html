Add and test your parts

In this activity, you will add your new parts to the CMM Inspection Programming device and machine databases and test them by loading them in a CMM inspection setup file.

  1. In a text editing program such as Notepad, open the device database file.

    [install directory]\Siemens\NX x.x\CMM_INSPECTION\resource\library\device\ascii\insp_device_database.dat

  2. In this window, click this step, select and copy the following line, and paste it as the last line in the file.

    DATA |Renishaw_PH10M_tr|3 |CMM Renishaw PH10M for Tool Racks| HEAD

  3. Save and close the device database file.

  4. Open the machine database file.

    [install directory]\Siemens\NX x.x\CMM_INSPECTION\resource\library\machine\ascii\insp_machine_database.dat

  5. In this window, click this step, select and copy the following line, and paste it as the last line in the file. 

    DATA | sheffield_rl_50_tr       | 5 | 3 Axis gantry CMM with tool rack | DMIS | Sheffield_tr| ${UGII_INSPECTION_POST_DIR}cmm_dmis_plugin.dat | 1.000000 | ${UGII_INSPECTION_LIBRARY_INSTALLED_MACHINES_DIR}sheffield_rl_50_tr\graphics\sheffield_rl_50_tr

  6. Save and close the machine database file.

  7. Start NX, open a work part, create an inspection setup file (File→New→Inspection tab→DMIS General Setup), and load your new machine, probe head, and at least two probe tools.

    The probe tools load into the tool rack in the order that they were selected.

  8. Create inspection features and then create inspection paths that use different probes.

    When you run simulations, the machine should select the tools using the entry and exit junctions you defined.

    Note

    If you encounter problems, in the Machine Tool Navigator, verify that your settings are correct.

    Required settings:

    • Part folders and parts are renamed correctly.

    • HEAD_A in Renishaw_PH10M_tr.prt has no classification.

    • Your tool rack is classified as a _DEVICE.

    • Your tool rack pockets and junctions are named correctly, and each pocket is classified as a _STATIC_HOLDER.

    • You loaded the correct _tr machine and head when testing.

Note

To use an actual changing rack on an actual CMM, you must configure your CMM execution software, or insert a CMM command such as the following DMIS statement into your program.

TH(Rack1)=THLDEF/SS(Tool1),1,SS(Tool2),2,SS(Tool3),3
