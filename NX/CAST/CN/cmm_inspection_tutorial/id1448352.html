Understanding your header information

  • UNITS/MM,ANGDEC,TEMPC indicates that in the program:

    • The units of measurement are in millimeters. The units information is based on the part’s units of measurement when the inspection setup file was created.

    • Angles are described in degree decimal form.

    • Temperature units are in Centigrade, as you specified in the Program Units group in the Program Header dialog box.

  • PN(PART_HDR_INFO_ID)=PARTID/'4HOLE_PMI' indicates the current part name. This name is based on the part name that you specified in the Part Identifier group in the Program Header dialog box.

  • PR(PART_HDR_INFO_RV)=PARTRV/'A' indicates the part revision method that you specified in the Part Identifier group in the Program Header dialog box.

  • For R(dt)=REPORT/DATE,TIME, R(ag)=REPORT/ALGOR, and R(tc)=REPORT/TEMPC:

    • R is a command that places additional information in the machine’s output file when the program runs on a CMM.

    • dt is a variable that lets the CMM report the date and time that the program is run.

    • ag is a variable that contains the type of algorithm that is used in calculating features when the CMM runs the program.

    • tc is a variable that contains the machine temperature, in degrees Celsius, when the CMM runs the program.

  • OUTPUT/R(dt), OUTPUT/R(ag), and OUTPUT/R(tc) output the variables defined above to the machine’s output file when the program runs.

  • PRCOMP/ON sets probe compensation to “ON”.

  • For SNSET/APPRCH,6.000000, SNSET/RETRCT,8.000000, SNSET/SEARCH, 25.000000, and SNSET/DEPTH, 10.00000:

    • The SNSET command defines and activates CMM sensor settings.

    • APPRCH sets the distance from the probe to the first feature to be measured.

    • RETRCT sets the distance that the probe retracts from the last feature to be measured.

    • SEARCH sets the distance that a probe continues measuring features beyond the APPRCH distance. If no feature is found, the program retracts the probe to its initial search point and returns an error.

    • DEPTH sets the distance that the probe penetrates into a feature along that feature's orientation vector.

Most code below these lines consists of sensor data, probe angles, and nominal point locations for each feature you linked to PMI.
