Align the part to the primary datum

In this activity, you will align your PCS Z normal with the part normal using a measured plane.

  1. In the Inspection Navigator, from the FEATURES program group, select TOP_PLANE_A to the PART_ALIGNMENT program group and drag it below OPER_MSG_BEGIN_INSP.

    Note that A signifies that the plane was used as the part’s primary datum.

  3. Double-click PATH_TOP_PLANE_A.

    The automatically generated path is set to measure nine points on the plane.

    In the real world, the plane’s surface is not perfectly aligned with the CMM table. While normal vector values for the CMM table would be very close to 0.000, 0.000, 1.000, the plane would have an actual normal vector such as 0.028987, -0.004567, 0.987654 (unitized), which is a compound angle. Therefore, aligning the Z axis of the PCS to the actual plane vector requires three alignment commands: two rotations and one translation.

    • The first command will rotate the PCS about its X-axis to align its Z vector to that of the actual plane. This defines the first half of the compound angle.

    • The second command will rotate the PCS about its Y-Axis to align its Z vector again. This defines the second half of the compound angle.

    • After the PCS Z vector is aligned to the actual normal vector of the plane, the third command will translate the Z origin of the PCS to that of the actual plane.

  4. In the Inspection Path dialog box, click OK.

  5. Using the plane feature as a measured reference, create a rotation alignment that rotates the PCS about its X-axis to begin aligning its Z vector to that of the actual plane.

    1. On the Insert toolbar, click Alignment .

    2. In the Alignment dialog box, from the Type list, select Rotate PCS.

    3. In the Rotation Parameters group, from the Method list, select Measured Feature.

    4. From the Rotation Axis list, select X-axis.

    5. From the Alignment Direction list, select +ZDIR.

    6. From the Select Feature list, select TOP_PLANE_A.

    7. In the PCS Name box, type PCS_AX.

    8. In the Operation Name box, type ROT_ALIGN_AX and click OK.

    The PCS now rotates about the X-axis to begin aligning the measured plane normal with that of the machine table.

  6. Using the plane feature as a measured reference, create another rotation alignment that rotates the PCS about its Y-axis to finish aligning its Z normal with that of the machine.

    1. In Inspection Navigator, in the PART_ALIGNMENT program group, right-click ROT_ALIGN_AX and choose Copy.

    2. Right-click ROT_ALIGN_AX again and choose Paste.

    3. Double-click ROT_ALIGN_AX_COPY.

    4. In the Alignment dialog box, in the Rotation Parameters group, from the Rotation Axis list, select Y-axis.

    5. In the PCS Name box, type PCS_AY.

    6. In the Operation Name box, type ROT_ALIGN_AY, and click OK.

    The Z normal of the PCS is now aligned with that of the machine table.

  7. Translate the PCS to the Z 0 origin of the measured plane.

    1. On the Insert toolbar, click Alignment .

    2. In the Alignment dialog box, from the Type list, select Translate PCS Origin.

    3. In the ZP group, from the Method list, choose Measured Feature.

    4. In the Select Feature list, select TOP_PLANE_A.

    5. In the PCS Name box, type PCS_TRN_ZORG.

    6. In the Operation Name box, type INSP_ALIGN_TZ, and click OK.

The part’s first three degrees of freedom are now constrained and the part is oriented to the machine table.
