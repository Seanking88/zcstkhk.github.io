JT attributes in NX

What is it?

The way properties are read from JT files into NX has changed.

  • NX now creates attributes on the NX part or on the solid bodies within the part depending on the corresponding JT object on which the properties were defined.

  • NX also names the NX objects and any corresponding features based on the names of the objects in the JT file.

Attributes that are defined in the JT files and that are not meant to be modified in NX, are called master data. NX identifies these attributes by the prefix specified by the Master Data Prefix customer default. All the attributes defined with this prefix in the JT file are locked in NX for modification. You can change this prefix to match the prefix used to define the master data in the JT file.

Where do I find it?

Menu

File→Utilities→Customer Defaults

Location in dialog box

Gateway→JT Files→Import tab
